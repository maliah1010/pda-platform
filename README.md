## Overview

The PDA Platform provides data infrastructure and AI tooling for project delivery assurance. It is designed for UK government Programme Management Offices (PMOs), Senior Responsible Owners (SROs), and project delivery professionals who need to assess project health, track compliance against the NISTA Programme and Project Data Standard, and run structured assurance processes through AI-powered MCP servers. The platform exposes 89 tools across 13 MCP modules, enabling Claude and other MCP-compatible AI assistants to parse project data, validate compliance, run risk analysis, track financial performance, manage benefits, and deliver gate-ready assurance assessments.

## Limitations

NISTA compliance scores generated by this tool are indicative assessments against the trial standard and do not constitute formal certification. This tool is provided as-is under the MIT licence; see LICENSE for full warranty and liability terms. The paper this tool accompanies describes the intended scope and known gaps: https://doi.org/10.5281/zenodo.18711384

## Key Features

- 89 MCP tools across 13 modules for AI-powered project delivery assurance
- 19 capabilities (P1–P19) covering artefact currency, compliance tracking, risk registers, financial management, gate readiness, and more
- 8 project data format parsers: MSPDI, Primavera P6, Jira, Monday, Asana, Smartsheet, GMPP, NISTA
- AI-powered narrative generation with confidence scoring via multi-sample consensus
- NISTA compliance tracking and GMPP quarterly reporting
- Gate readiness assessment with 8-dimension scoring and IPA gate-specific weighting
- Agent Readiness Maturity Model (ARMM) for AI deployment governance
- Assumption drift tracking with cascade impact analysis

All modules are served through a single unified endpoint (`pda-platform-server`) or individually. A remote variant (`pda-platform-remote`) adds SSE transport for use with Claude.ai.

## Assurance Capabilities

| ID | Capability | Description |
|----|-----------|-------------|
| P1 | Artefact Currency Validator | Detects stale or anomalously refreshed evidence artefacts against gate deadlines |
| P2 | Longitudinal Compliance Tracker | Persists NISTA compliance scores and surfaces trend direction and threshold breaches |
| P3 | Cross-Cycle Finding Analyzer | Extracts review actions, deduplicates within cycles, and detects cross-cycle recurrence |
| P4 | Confidence Divergence Monitor | Detects when AI extraction samples diverge or consensus declines across review cycles |
| P5 | Adaptive Review Scheduler | Recommends optimal review timing based on P1-P4 signals rather than fixed calendars |
| P6 | Override Decision Logger | Structured logging and pattern analysis of governance override decisions |
| P7 | Lessons Learned Knowledge Engine | Ingests structured lessons and provides keyword and semantic search |
| P8 | Assurance Overhead Optimiser | Tracks assurance effort, identifies waste, and measures confidence outcomes |
| P9 | Assurance Workflow Engine | Deterministic multi-step orchestrator sequencing P1-P8 with inter-step data flow |
| P10 | Project Domain Classifier | Classifies projects into complexity domains with tailored assurance profiles |
| P11 | Assumption Drift Tracker | Monitors assumption drift against baselines with cascade impact analysis |
| P12 | Agent Readiness Maturity Model | Four-dimension, 28-topic, 251-criterion maturity assessment for AI agent deployment |
| P13 | Benefits Realisation Management | IPA/Green Book-compliant benefits register, measurement tracking, dependency network |
| P14 | Gate Readiness Assessor | Synthesises P1-P13 outputs into gate-specific readiness scores with IPA weighting |

## Connect to PDA Platform

PDA Platform works with Claude, ChatGPT, Gemini, and any MCP-compatible client. **89 tools, one connection.**

**Quickest start — Claude Desktop:**

```bash
pip install pda-platform
```

Add to your Claude Desktop config (`~/Library/Application Support/Claude/claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "pda-platform": {
      "command": "pda-platform-server",
      "args": []
    }
  }
}
```

**Remote — no install needed:**

Connect any MCP client to the SSE endpoint: `https://pda-platform-i33p.onrender.com/sse`

**Host your own:**

[![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy?repo=https://github.com/antnewman/pda-platform)

See the full **[Connection Guide](docs/connection-guide.md)** for Claude.ai, ChatGPT, Gemini, and other clients.

## Quick Start (Library)

### Install

```bash
# Install everything in one command
pip install pda-platform

# Or install individual packages
pip install pm-data-tools        # core library only
pip install pm-mcp-servers       # MCP servers only
pip install agent-task-planning  # AI reliability framework only
```

### Configure MCP

Add the PDA Platform server to your Claude Desktop configuration (`claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "pda-platform": {
      "command": "pda-platform-server",
      "args": []
    }
  }
}
```

### Parse project data

```python
from pm_data_tools import parse_project

project = parse_project("schedule.mpp")
```

### Validate NISTA compliance

```python
from pm_data_tools.validators import NISTAValidator

result = NISTAValidator().validate(project)
print(f"Compliance: {result.compliance_score}%")
```

### Run a gate readiness assessment

```python
from pm_data_tools.assurance.gate_readiness import GateReadinessAssessor, GateType
from pm_data_tools.db.store import AssuranceStore

store = AssuranceStore()
assessor = GateReadinessAssessor(store=store)

result = assessor.assess("PROJ-001", GateType.GATE_3)
print(f"Readiness: {result.readiness.value}")
print(f"Score: {result.composite_score:.0%}")
```
